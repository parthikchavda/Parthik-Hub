📘 ડિજિટલ ઇલેક્ટ્રોનિક્સ, VLSI & FPGA
સંપૂર્ણ વિષયવાર માર્ગદર્શિકા (ગુજરાતી)

MCQs આધારિત + વધારાની માહિતી | સ્પર્ધાત્મક પરીક્ષા, યુનિવર્સિટી અને ઇન્ટરવ્યુ માટે ઉપયોગી

૧. સંખ્યા પદ્ધતિ (Number System)

સંખ્યા પદ્ધતિઆધાર (Base)વપરાતા અંકો/અક્ષરોઉદાહરણ
દ્વિમાન (Binary)20, 1(1011)₂ = 11₁₀
દશમાન (Decimal)100-925₁₀
અષ્ટમાન (Octal)80-7(17)₈ = 15₁₀
ષોડશમાન (Hexadecimal)160-9, A-F(1A)₁₆ = 26₁₀
✍️ વધારાની માહિતી:
  • દ્વિમાન → દશમાન: દરેક બીટને 2^n વડે ગુણો (n = સ્થાન, જમણેથી 0).
  • દશમાન → દ્વિમાન: 2 વડે વારંવાર ભાગાકાર, બાકીના અંક ઊલટા ક્રમે લખો.
  • ષોડશમાન → દ્વિમાન: દરેક ષોડશમાન અંકને 4-બીટ દ્વિમાનમાં ફેરવો.
  • દ્વિમાન → અષ્ટમાન: 3-3 બીટના જૂથ બનાવો (જમણી બાજુથી).

૨. બુલિયન બીજગણિત (Boolean Algebra)

નિયમ (Law)સૂત્ર
સ્વતઃસિદ્ધ (Identity)A + 0 = A,   A·1 = A
પૂરક (Complement)A + A' = 1,   A·A' = 0
સ્વસમ (Idempotent)A + A = A,   A·A = A
સહચારી (Associative)(A+B)+C = A+(B+C)
વિતરક (Distributive)A(B+C) = AB + AC
શોષણ (Absorption)A + AB = A,   A(A+B) = A
ડી મોર્ગન (De Morgan)(A·B)' = A' + B',   (A+B)' = A'·B'
🔹 મિનટર્મ (Minterm): ચલોનું AND, દરેક ચલ તેના મૂળ અથવા પૂરક સ્વરૂપમાં.
🔹 મેક્સટર્મ (Maxterm): ચલોનું OR.
🔹 SOP (Sum of Products): મિનટર્મોનો OR.
🔹 POS (Product of Sums): મેક્સટર્મોનો AND.

૩. લોજિક ગેટ્સ (Logic Gates)

ગેટઆઉટપુટ સ્થિતિ (HIGH માટે)
ANDબધા ઇનપુટ = 1
ORકોઈપણ એક ઇનપુટ = 1
NOT (Inverter)ઇનપુટની વિરુદ્ધ
NANDબધા ઇનપુટ 1 હોય તો જ 0, બાકી 1
NORબધા ઇનપુટ 0 હોય તો જ 1, બાકી 0
XORઇનપુટ અલગ-અલગ હોય (01 અથવા 10)
XNORઇનપુટ સમાન હોય (00 અથવા 11)
🌟 Universal Gates: NAND અને NOR – કોઈપણ લોજિક ગેટ ફક્ત આ બે ગેટ વડે બનાવી શકાય છે.
🌟 Fan-out: એક ગેટના આઉટપુટ વડે કેટલા બીજા ગેટ ચલાવી શકાય છે.

૪. K-Map અને સરળીકરણ (Karnaugh Map & Simplification)

  • K-Map: બુલિયન સમીકરણો ગ્રાફિકલ રીતે સરળ બનાવવા માટે.
  • સેલની સંખ્યા = 2ⁿ (n = ચલોની સંખ્યા).
  • ગ્રુપિંગ નિયમો:
    • 2 સેલ → 1 ચલ દૂર થાય
    • 4 સેલ → 2 ચલ દૂર થાય
    • 8 સેલ → 3 ચલ દૂર થાય
  • Don't Care (X): મોટા ગ્રુપ બનાવવા માટે ઉપયોગ કરી શકાય.
  • Quine-McCluskey Method: 5-6 કરતાં વધુ ચલો હોય ત્યારે K-Map ને બદલે આ પદ્ધતિ વપરાય છે.

૫. એડર અને સબ્ટ્રેક્ટર (Adder & Subtractor)

✧ હાફ એડર (Half Adder)

  • 2 ઇનપુટ (A, B) → 2 આઉટપુટ: Sum = A⊕B, Carry = A·B

✧ ફુલ એડર (Full Adder)

  • 3 ઇનપુટ (A, B, Cin) → Sum, Carry-out → બે હાફ એડર વડે બનાવી શકાય.

✧ સબ્ટ્રેક્ટર

  • Half Subtractor: Difference = A⊕B, Borrow = A'·B
  • Full Subtractor: 3 ઇનપુટ, 2 આઉટપુટ

✧ Carry Look-Ahead Adder (CLA)

  • રિપલ કેરી એડરમાં કેરી પ્રસારણમાં વિલંબ થાય છે. CLA એડરમાં કેરી અગાઉથી જનરેટ થાય છે → ઝડપી.

✧ 2's Complement વડે બાદબાકી

  • સબ્ટ્રાહેન્ડનો 2’s complement લઈને મિન્યુએન્ડમાં ઉમેરો.

૬. કોડ કન્વર્ઝન (Code Conversion)

કોડલાક્ષણિકતા

🔁 રૂપાંતર:
Binary → Gray: MSB સરખો રાખો, બાકી દરેક બીટ = અગાઉની બીટ XOR હાલની બીટ.
Gray → Binary: MSB સરખો, પછી દરેક નવી બીટ = અગાઉની Binary XOR હાલની Gray.

૭. કોમ્બિનેશનલ સર્કિટ્સ (MUX, DEMUX, ડીકોડર, એન્કોડર, કંપેરેટર)

  • Multiplexer (MUX): Data Selector – n સિલેક્ટ લાઈન → 2ⁿ ઇનપુટ.
  • Demultiplexer (DEMUX): Data Distributor – એક ઇનપુટને અનેક આઉટપુટમાં મોકલે.
  • Decoder: n ઇનપુટ → 2ⁿ આઉટપુટ (ફક્ત એક આઉટપુટ HIGH). Enable pin વડે સક્રિય/નિષ્ક્રિય.
  • Encoder: 2ⁿ ઇનપુટ → n આઉટપુટ. Priority Encoder: એકસાથે બહુવિધ ઇનપુટ હોય તો ઉચ્ચ પ્રાથમિકતાવાળાને પ્રતિસાદ આપે.

૯. સિક્વેન્શિયલ સર્કિટ્સ – ફ્લિપ-ફ્લોપ્સ

ફ્લિપ-ફ્લોપઇનપુટલાક્ષણિકતા સમીકરણવિશેષ
SRS, RQ⁺ = S + R'QS=R=1 અમાન્ય
DDQ⁺ = DData / Delay FF
JKJ, KQ⁺ = JQ' + K'QJ=K=1 → Toggle (Universal FF)
TTQ⁺ = T ⊕ QT=1 → Toggle (Divide‑by‑2)
⚠️ Race Around Condition: JK FF માં J=K=1 અને level‑triggered હોય ત્યારે થાય.
✅ Master‑Slave FF: બે FFની મદદથી race condition દૂર થાય.
✅ Edge Triggered: ઘડિયાળના Rising/Falling એજ પર જ સ્થિતિ બદલાય.
✅ Preset (Q=1), Clear (Q=0): અસિંક્રોનસ ફોર્સિંગ.

૧૦. કાઉન્ટર્સ (Counters)

કાઉન્ટર પ્રકારલાક્ષણિકતા
Ripple (Asynchronous)દરેક FFનું ક્લોક પાછલા FFના આઉટપુટ પર – ધીમું (propagation delay)
Synchronousબધા FFને એક સાથે ક્લોક – ઝડપી
Ring CounterOne‑hot code – 4-bit માં 4 સ્થિતિ
Johnson CounterTwisted ring – 4-bit માં 8 સ્થિતિ
Decade Counter (Mod‑10)0 થી 9 સુધી ગણે (BCD)
Up‑Down Counterઉપર અને નીચે બંને ગણી શકે
🔢 Modulus = કાઉન્ટરની કુલ સ્થિતિઓની સંખ્યા.
🔢 n-bit counter મહત્તમ ગણતરી = 2ⁿ − 1.
🔄 Reset: કાઉન્ટરને 0 પર લાવે છે.

૧૨. લોજિક ફેમિલી (Logic Families)

ફેમિલીટેક્નોલોજીઝડપપાવરનોઇઝ ઇમ્યુનિટી
TTLBJTમધ્યમવધુમધ્યમ
CMOSNMOS+PMOSમધ્યમખૂબ ઓછોઊંચી
ECLEmitter Coupled BJTસૌથી ઝડપીખૂબ વધુનીચી
💡 Fan-out: TTL ~10, CMOS >50.
💡 Power‑Delay Product: ઊર્જા કાર્યક્ષમતા માપવા માટે.
💡 CMOS Static Power: આદર્શ રીતે શૂન્ય (માત્ર leakage).

૧૩. ADC અને DAC (Converters)

◆ DAC (Digital to Analog)

  • Weighted Resistor DAC: ઊંચા બીટ્સ માટે રેઝિસ્ટર વેલ્યુમાં ઘણો તફાવત – જટિલ.
  • R-2R Ladder DAC: માત્ર બે રેઝિસ્ટર વેલ્યુ (R અને 2R) – સરળ અને ચોક્કસ.
  • Full Scale Output: જ્યારે બધા બીટ્સ = 1.
  • Resolution = 1/2ⁿ (Vref ની તુલનામાં)

◆ ADC (Analog to Digital)

  • Flash ADC: સૌથી ઝડપી, પણ 2ⁿ કંપેરેટર – ખર્ચાળ.
  • SAR ADC: બાઈનરી સર્ચ પદ્ધતિ – મધ્યમ ઝડપ, સારું સંતુલન.
  • Dual Slope ADC: ધીમું પણ નોઇઝ પ્રતિરોધક – માપન માટે સારું.
  • Counter Type ADC: ધીમું, સરળ.
📏 Quantization Error: ±0.5 LSB.
🎯 Sampling Theorem: fs ≥ 2 × fmax (એલિયાસિંગ ટાળવા).
⚠️ Aliasing: ઓછું સેમ્પલિંગ થાય ત્યારે ઉચ્ચ આવૃત્તિના સિગ્નલ નીચી આવૃત્તિ તરીકે દેખાય.

૧૪. VLSI & MOSFET

✦ CMOS Inverter

  • Pull‑up: PMOS (VDD તરફ), Pull‑down: NMOS (GND તરફ).
  • Switching Threshold ≈ VDD/2 (જ્યારે Vin = Vout).
  • Dynamic Power = C × V² × f
  • Static Power: Leakage current (subthreshold, gate leakage).

✦ MOSFET Operating Regions

  • Cutoff: VGS < Vth → સ્વીચ OFF
  • Triode (Linear): VGS > Vth, VDS < VGS-Vth → સ્વીચ ON
  • Saturation: VDS ≥ VGS-Vth → એમ્પ્લીફાયર મોડ, ડિજિટલ લોજિકમાં ચાર્જિંગ.

✦ Short Channel Effects

  • DIBL (Drain Induced Barrier Lowering): Vth ઘટે છે.
  • Velocity Saturation: કરંટ પર મર્યાદા.
  • Body Effect: VSB વધે તો Vth વધે.
📈 Moore's Law: ટ્રાન્ઝિસ્ટરની સંખ્યા દર 18-24 મહિને બમણી થાય.
🔬 Lithography: UV પ્રકાશ વડે પેટર્ન ટ્રાન્સફર.
🔌 Copper Interconnect: એલ્યુમિનિયમ કરતાં ઓછો પ્રતિકાર, વધુ સારી કાર્યક્ષમતા.

૧૫. FPGA અને VHDL

🧩 FPGA Architecture

  • CLB (Configurable Logic Block): LUT + Flip‑Flop.
  • LUT (Look‑Up Table): નાનું ROM – કોઈપણ બુલિયન ફંક્શન.
  • BRAM: Block RAM (ઑન‑ચિપ મેમરી).
  • DSP Slice: Multiply‑Accumulate (MAC) માટે હાર્ડવેર.

📝 VHDL Basics

  • Entity: બાહ્ય ઇન્ટરફેસ (ports).
  • Architecture: આંતરિક વર્તણૂક.
  • Process Block: અંદર સિક્વેન્શિયલ કોડ.
  • Signal Assignment: `<=`
  • Data Types: bit, std_logic, std_logic_vector, integer.
  • Sensitivity List: પ્રોસેસ ક્યારે ચાલે તે નક્કી કરે.
  • rising_edge(clk): positive edge detection.
🧪 Testbench: HDL કોડ ચકાસવા માટે.
⚙️ Synthesis: VHDL → Gate‑level netlist.
📁 Bitstream: FPGA પ્રોગ્રામ કરવા માટેની ફાઇલ.

૧૬. MCQમાં ન હોય તેવા મહત્વપૂર્ણ વિષયો (વધારાની માહિતી)

  • Mealy vs Moore Machine: Mealy – આઉટપુટ = f(ઇનપુટ, પ્રેઝન્ટ સ્ટેટ) ; Moore – આઉટપુટ = f(પ્રેઝન્ટ સ્ટેટ). Mealyમાં ઓછા સ્ટેટ.
  • Metastability: ફ્લિપ-ફ્લોપ અનિશ્ચિત સ્થિતિ – ઉકેલ: બે FFનું સિંક્રોનાઈઝર.
  • Clock Skew & Jitter: Skew = સમયનો તફાવત; Jitter = કાંઠાની સ્થિતિમાં ફેરફાર.
  • Power Gating: સ્લીપ ટ્રાન્ઝિસ્ટર વડે બ્લોકનો પાવર કાપવો.
  • Clock Gating: જ્યારે જરૂર ન હોય ત્યારે ક્લોક બંધ કરવી.
  • Verilog HDL: VHDL જેવી બીજી HDL – always @(posedge clk).
  • FinFET Technology: ચેનલ પર ત્રણ બાજુથી ગેટ – 22nm અને નાના નોડમાં ઉપયોગ, લીકેજ ઓછું.
  • FIFO (First In First Out): ડેટા બફરિંગ – Full/Empty flags.
