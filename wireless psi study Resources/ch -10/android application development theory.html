Android Application Development — Complete Theory

🧠 1. Android OS – Core Concepts

Android: Open-source mobile OS based on Linux kernel. Developed by Android Inc. (acquired by Google 2005). First device: HTC Dream (2008).

  • Open Source (AOSP): Free to modify, manufacturers customize (Samsung One UI, etc.)
  • Sandbox security: Each app runs isolated with unique UID.
  • ART runtime (Android 5.0+) replaces Dalvik – Hybrid AOT+JIT compilation → faster startup, better battery.
  • Key versions: Marshmallow (runtime permissions), Lollipop (Material Design), Android 10 (dark theme), Android 12 (Material You).

📌 Exam tip Dalvik used JIT, ART uses AOT+JIT; APK = Android Package Kit.

🏗️ 2. Android Architecture (5 Layers)
Layer Components / Role
Linux Kernel Device drivers, memory/power management, Binder IPC, Low Memory Killer, security (UID sandbox).
HAL Hardware Abstraction Layer – standard interface for camera, sensors; hardware independent.
Native Libraries & ART SQLite, WebKit, OpenGL, Media Framework + Android Runtime (executes DEX bytecode).
Application Framework Activity Manager, Window Manager, Content Providers, View System, Notification Manager – APIs for developers.
Applications System apps (Phone, Contacts) + 3rd party apps (your app).

Binder Driver – high-performance IPC. Activity Manager manages lifecycle & back stack.

🖥️ 3. Activities & UI Design

Activity = single screen with UI. Lifecycle (must memorize order): onCreate() → onStart() → onResume() → onPause() → onStop() → onDestroy().

  • onCreate(): setContentView() , initialize UI.
  • onPause(): save transient data.
  • onSaveInstanceState() handles rotation data loss.

Intent: Explicit (target component) & Implicit (action like VIEW, SEND). UI building: XML layouts (LinearLayout, RelativeLayout, ConstraintLayout). Units: dp (size), sp (text).

<TextView android:layout_width="wrap_content" android:text="Hello"/>
✨ 4. Advanced UI: Fragments, RecyclerView, Styles

Fragment – modular UI block inside Activity; has own lifecycle (onCreateView, onViewCreated). Reused for tablet/phone adaptive layouts.

RecyclerView – high-performance lists with ViewHolder pattern & LayoutManager (Linear/Grid). View recycling boosts scrolling.

Styles & Themes: Style = attribute collection for a View; Theme = whole app/activity (Material Components). Dark mode via DayNight theme.

Animations: View animations (fade, slide) and Property animations (actual property change).

🍞 5. Toast · Menu · Dialog

Toast: short popup, auto-disappear. Toast.makeText(context, msg, LENGTH_SHORT).show().

Menu types: Options Menu (action bar), Context Menu (long-press), Popup Menu. Inflate from XML.

Dialog: modal window – AlertDialog (confirmation), DatePickerDialog, Custom Dialog using builder.setView(). User must interact before returning.

new AlertDialog.Builder(this)
    .setTitle("Confirm")
    .setMessage("Delete?")
    .setPositiveButton("Yes", (d,w)->{})
    .show();
📋 6. Lists & Adapters (ListView / RecyclerView)

Adapter – bridge between data and UI. ArrayAdapter (simple lists), CursorAdapter (SQLite), BaseAdapter (custom).

ViewHolder pattern – caches view references inside getView() / onBindViewHolder() to avoid expensive findViewById calls → smooth scrolling.

RecyclerView.Adapter mandatory ViewHolder + LayoutManager. Notify dataset changes: notifyDataSetChanged() or specific notifyItemInserted().

🗄️ 7. Working with SQLite Database

SQLite: embedded, serverless, zero-config. SQLiteOpenHelper handles DB creation/upgrades.

CRUD operations: insert(), query() / rawQuery(), update(), delete(). Cursor to iterate results (always close).

// Insert example
ContentValues cv = new ContentValues();
cv.put("name", "Raj");
db.insert("students", null, cv);

// Query
Cursor c = db.rawQuery("SELECT * FROM students", null);
while(c.moveToNext()) { String name = c.getString(0); }
c.close();

Combine with ListView/RecyclerView using CursorAdapter or convert to list + ArrayAdapter.

📖 Quick Revision Summary
  • Activity lifecycle order → important MCQ.
  • Toast vs Dialog: Toast non-interactive, Dialog modal.
  • Android Architecture: Kernel → HAL → Libraries/ART → Framework → Apps.
  • Intent types: Explicit & Implicit.
  • RecyclerView = modern, ViewHolder mandatory, LayoutManager controls orientation.
  • SQLiteOpenHelper: onCreate() for table creation, onUpgrade() for version changes.
