Runtime: Dalvik vs ART EXAM FAVE

Dalvik VM was the original Android runtime (JIT — Just In Time compilation).

ART (Android Runtime) replaced Dalvik starting with Lollipop (API 21) and uses AOT (Ahead Of Time) compilation — faster execution, better performance.

Dalvik
JIT compilation — older Android
ART
AOT compilation — Android 5.0+ default

Process Priority (Low memory kill order)

Android kills processes to free RAM. Kill order (first to last):

1. Background Process 2. Empty Process 3. Service Process 4. Visible Process 5. Foreground Process (last)
⚠️ Background process is killed FIRST when RAM is low. Foreground process (user-active app) is killed LAST.

Android Architecture (Layered Stack)
Android આર્કિટેક્ચર — સ્તરીય સ્ટૅક

6 Layers — Top to Bottom EXAM CRITICAL

Android follows a layered architecture. Remember: Apps → Framework → Runtime+Libraries → HAL → Kernel

① Applications Layer (Top — closest to user)
Browser, Contacts, Camera, Phone, Email, Custom Apps
② Application Framework Layer
Activity Manager · Window Manager · Content Providers · View System · Package Manager · Notification Manager · Resource Manager · Location Manager · Telephony Manager · Camera Services
③ Android Runtime (ART / Dalvik)
Core Libraries + ART Runtime (AOT compilation)
④ Native C/C++ Libraries
SQLite · WebKit · OpenGL ES · Media Framework · Surface Manager · SSL · Skia (2D graphics) · libc
⑤ Hardware Abstraction Layer (HAL)
Camera HAL · Bluetooth HAL · Audio HAL · Sensor HAL — bridges hardware to framework
⑥ Linux Kernel (Bottom — closest to hardware)
Device Drivers · Memory Mgmt · Process Scheduling · Networking Stack · Binder IPC · Power Management · Security · Thread Management · Bluetooth/Audio/Sensor drivers
💡 Exam: "Which layer is at BOTTOM?" → Linux Kernel. "Which is at TOP?" → Applications. "Which contains SQLite/WebKit?" → Libraries (C/C++).

HAL — Hardware Abstraction Layer

HAL stands for Hardware Abstraction Layer. It provides a standard interface that lets the Android framework interact with hardware-specific drivers without knowing hardware details.

ℹ️ HAL bridges the gap between hardware (Camera, Bluetooth, Sensors) and the Android Framework. Manufacturers implement HAL for their specific hardware.

Sandbox Security

Each Android app runs in its own sandbox — an isolated process with a unique Linux user ID. Apps cannot access each other's data directly. This is enforced at the kernel level.

The AndroidManifest.xml declares permissions. Since Android 6.0 Marshmallow, dangerous permissions require runtime user approval.

Each Method — Detailed Explanation

MethodWhen CalledWhat to Do Here
onCreate() FIRST time activity is created. Called only once per lifecycle. Initialize UI (setContentView()), findViewById(), setup variables, initialize data
onStart() Activity becomes VISIBLE to user (after onCreate or onRestart) Register broadcast receivers, load fresh data for display
onResume() Activity is in FOREGROUND and INTERACTIVE (highest priority state) Start animations, camera, GPS, resume paused operations
onPause() Activity partially hidden (another activity on top, dialog shown). Activity is still VISIBLE. Pause animations, save unsaved data, release camera/GPS
onStop() Activity is COMPLETELY HIDDEN (user navigated to another app) Stop resource-heavy operations, save state to database
onRestart() Called BEFORE onStart() when activity returns from STOPPED state Refresh data that may have changed while stopped
onDestroy() Activity is being DESTROYED (user pressed back, or system killed it) Release all resources, close DB connections
onSaveInstanceState() Called before activity may be destroyed (rotation, system kill) Save UI state (EditText values, scroll position) into Bundle
onRestoreInstanceState() After onCreate/onStart when saved state exists Restore previously saved UI state from Bundle

State Priorities

Resumed (Active) — HIGHEST PRIORITY Paused (Partially visible) Stopped (Invisible) Destroyed — LOWEST PRIORITY

Intents & Navigation
Intents અને Navigation

Intent — What is it?

An Intent is a messaging object used to request an action from another component. It is used to start activities, start services, or deliver broadcasts.

Explicit Intent
Targets a specific component by class name
Implicit Intent
Declares general action; system finds best match
// Explicit — start specific activity + pass data Intent intent = new Intent(this, SecondActivity.class); intent.putExtra("username", "Raj"); startActivity(intent); // Get data in target activity String name = getIntent().getStringExtra("username"); // Implicit — share/open URL Intent intent = new Intent(Intent.ACTION_VIEW); intent.setData(Uri.parse("https://google.com")); startActivity(intent);

What is Toast? EXAM TOPIC

A Toast is a small popup message shown briefly at the bottom of the screen. It is lifecycle-independent — it does not block user interaction with the background.

Blocks interaction?
❌ No (non-modal)
Disappears automatically?
✅ Yes
Can have action button?
❌ No (use Snackbar for that)
Better alternative
Snackbar (supports action)

Toast — All Methods

// Create and show Toast Toast.makeText(context, "Hello World!", Toast.LENGTH_SHORT).show(); Toast.makeText(context, "Longer message", Toast.LENGTH_LONG).show(); // Constants Toast.LENGTH_SHORT // ~2 seconds Toast.LENGTH_LONG // ~3.5 seconds // Cancel toast early Toast t = Toast.makeText(context, "msg", Toast.LENGTH_LONG); t.show(); t.cancel(); // removes toast before time ends // Set toast margin t.setMargin(0.1f, 0.1f); // setMargin(horizontalMargin, verticalMargin)

Room Persistence Library
Room — SQLite ઉપર Abstract Layer

Room — Modern SQLite Abstraction

Room is part of Android Architecture Components. It provides an abstraction layer over SQLite, eliminating boilerplate code. Follows the MVVM / Repository pattern.

// ① Entity (defines a table) @Entity(tableName = "users") public class User { @PrimaryKey(autoGenerate = true) public int id; public String name; public int age; } // ② DAO (Data Access Object) @Dao public interface UserDao { @Insert void insert(User user); @Query("SELECT * FROM users") List<User> getAll(); @Delete void delete(User user); } // ③ Database class @Database(entities = {User.class}, version = 1) public abstract class AppDatabase extends RoomDatabase { public abstract UserDao userDao(); } // Usage AppDatabase db = Room.databaseBuilder(context, AppDatabase.class, "mydb").build(); db.userDao().insert(new User("Raj", 25));
AnnotationPurpose
@EntityMarks class as database table / defines schema
@DaoMarks interface as DAO (data access object)
@DatabaseMarks class as the database definition
@QueryMarks a SELECT/other SQL query method
@InsertInsert operation
@DeleteDelete operation
@UpdateUpdate operation
