Abbreviations — Short Form → Full Form
ટૂંકા સ્વરૂપ → પૂર્ણ સ્વરૂપ
APKABBREVIATION
Full FormAndroid Package Kit
The file format (.apk) used to distribute and install Android apps. Like a ZIP file containing your app's code, resources, and manifest.
Android એપ ઇન્સ્ટોલ કરવા માટેનું ફાઇલ ફોર્મેટ. (.apk) — આ ફાઇલ play store કે manually install થાય છે.
ADBABBREVIATION
Full FormAndroid Debug Bridge
A command-line tool that lets you communicate with a connected Android device or emulator. Used to install APKs, view logs, run shell commands.
Android ડિવાઇસ/ઇમ્યુલેટર સાથે computer ને જોડવાનું ટૂલ. adb install, adb devices, adb uninstall — debug/test વખતે વપરાય.
Commandsadb devices | adb install app.apk | adb uninstall com.pkg
SDKABBREVIATION
Full FormSoftware Development Kit
A collection of tools, libraries, APIs, and documentation used to develop Android applications. Includes compilers, emulators, and build tools.
Android app બનાવવા માટેનો ટૂલ-સેટ. Libraries, compilers, emulator — બધું SDK માં. minSdkVersion manifest/gradle માં.
ARTABBREVIATION
Full FormAndroid Runtime
The runtime environment that executes Android apps. Replaced Dalvik in Android 5.0 (Lollipop). Uses AOT (Ahead-of-Time) compilation — compiles app code at install time for faster execution.
Android app ચલાવવાનું runtime environment. Lollipop (5.0) થી Dalvik ને replace કર્યું. AOT compilation = install time પર code compile — faster performance.
AOTABBREVIATION
Full FormAhead-Of-Time Compilation
Compilation technique where code is compiled BEFORE execution (at install time). ART uses AOT — makes apps run faster because machine code is ready.
Install time પર code compile થઈ જાય — execution પહેલાં. ART આ technique વાપરે. Faster performance.
JITABBREVIATION
Full FormJust-In-Time Compilation
Compilation technique where code is compiled DURING execution (at runtime). Dalvik VM used JIT — slower than AOT because compilation happens while running.
Running time (execution) દરમ્યાન code compile. Dalvik JIT વાપરતો. AOT કરતાં ધીમો — ART (AOT) એ replace કર્યો.
JVMABBREVIATION
Full FormJava Virtual Machine
The standard runtime for running Java bytecode on a computer. Android does NOT use JVM — it uses its own ART (previously Dalvik) instead.
Java code ચલાવવાની standard virtual machine. Android JVM વાપરતું નથી — ART/Dalvik (Android-specific) વાપરે.
HALABBREVIATION
Full FormHardware Abstraction Layer
A layer in Android architecture that provides a standard interface between Android framework and hardware-specific drivers (Camera, Bluetooth, Sensors). Allows hardware manufacturers to implement drivers without changing the framework.
Android framework અને hardware ની વચ્ચેનો layer. Camera, Bluetooth, Sensor ના drivers ને framework થી અલગ રાખે. Hardware maker HAL implement કરે.
IPCABBREVIATION
Full FormInter-Process Communication
Mechanism allowing separate processes (apps) to communicate with each other. Android uses "Binder" IPC (implemented in Linux kernel) for this purpose.
અલગ-અલગ processes (apps) વચ્ચે communication. Android Binder IPC (Linux Kernel) વાપરે.
UIABBREVIATION
Full FormUser Interface
The visual part of the app that the user sees and interacts with — buttons, text, images, layouts. In Android, built using XML layouts and View classes.
User ને દેખાતો app નો ભાગ — buttons, text, images. Android માં XML layout files + View classes વડે.
XMLABBREVIATION
Full FormeXtensible Markup Language
A markup language used in Android to define UI layouts, string resources, and the AndroidManifest. Human-readable, tag-based format like HTML.
Android UI layout, strings, manifest — બધા XML format માં. Tags (,
MVCABBREVIATION
Full FormModel-View-Controller
A software design pattern separating app logic into 3 parts: Model (data), View (UI), Controller (logic/input handling). Adapter pattern in Android lists follows MVC concept.
Software design pattern: Model (data) + View (UI) + Controller (logic). Android list adapter pattern MVC concept follow કરે.
MVVMABBREVIATION
Full FormModel-View-ViewModel
Modern Android architecture pattern. ViewModel holds UI data and survives rotation. Room library + LiveData follows MVVM / Repository pattern.
Modern Android architecture: Model + View + ViewModel. ViewModel rotation survive કરે. Room library MVVM follow કરે.
APIABBREVIATION
Full FormApplication Programming Interface
A set of rules/functions that allow apps to interact with the OS, other apps, or web services. Android framework provides APIs to developers. "API level" = Android version number.
Android framework developer ને APIs provide કરે. API level = Android version (API 21 = Android 5.0). Web services સાથે communicate.
dp / dipABBREVIATION
Full FormDensity-Independent Pixels
Unit for layout dimensions in Android. 1dp = 1 pixel on 160dpi screen. Scales automatically on different screen densities — layouts look consistent on all devices.
Layout dimensions (width, height, margin, padding) માટે unit. Different screen size/density પર consistent look. Layout માં dp વાપરો.
spABBREVIATION
Full FormScale-Independent Pixels
Unit for TEXT SIZE in Android. Like dp but also scales with the user's font size preference (accessibility). Always use sp for android:textSize.
Text size (android:textSize) માટે unit. User's font size setting સાથે scale. Text size = sp, Layout dimensions = dp.
SQLABBREVIATION
Full FormStructured Query Language
Language used to manage and query relational databases. Android uses SQLite which supports SQL. Commands: SELECT, INSERT, UPDATE, DELETE, CREATE TABLE, DROP TABLE.
Database manage/query કરવાની ભાષા. Android SQLite SQL support કરે. SELECT, INSERT, UPDATE, DELETE, CREATE TABLE, DROP TABLE.
DAOABBREVIATION
Full FormData Access Object
In Android Room library, DAO is an interface annotated with @Dao that defines database operations (INSERT, DELETE, @Query). Separates database logic from app logic.
Room library માં @Dao interface — database operations define. App logic ને DB logic થી અલગ. @Insert, @Delete, @Query annotations.
Code@Dao interface UserDao { @Query("SELECT * FROM users") List<User> getAll(); }
FABABBREVIATION
Full FormFloating Action Button
A circular Material Design button that floats above the UI, used for the primary action of a screen (like compose, add, create). Class: FloatingActionButton.
Screen ના primary action (add/create) માટે floating circular button. Material Design component. Class: FloatingActionButton.
CABABBREVIATION
Full FormContextual Action Bar
A temporary toolbar that replaces the Action Bar when items are selected (long-press). Used for bulk operations like delete, share. Managed by ActionMode class.
Item select (long-press) પર temporary toolbar. Bulk operations (delete, share) માટે. ActionMode class manage કરે.

Core Concepts
મૂળ ખ્યાલો
KernelCONCEPT
Core of an operating system. The kernel is the lowest-level software that directly manages hardware — CPU, memory, device drivers, file system, security. Android is based on the Linux kernel.
Operating system નો core (heart). Hardware — CPU, memory, drivers — directly manage. Android Linux kernel પર based. Architecture ના bottom layer.
SandboxCONCEPT
Isolated execution environment. Each Android app runs in its own sandbox — a separate process with a unique Linux user ID. Apps CANNOT access each other's data directly. Enforced by the kernel.
દરેક app ને isolated environment. Unique Linux user ID. Apps directly data share ન કરી શકે. Security kernel level enforce. App delete → data delete.
LifecycleCONCEPT
The sequence of states an Activity goes through from creation to destruction. States: Created → Started → Resumed → Paused → Stopped → Destroyed. Android system calls specific callback methods at each transition.
Activity ની create થઈ → destroy સુધીની journey. System automatically onCreate, onStart, onResume, onPause, onStop, onDestroy call. Resources manage.
IntentCONCEPT
A messaging object used to request an action from another component. Used to: start an Activity, start a Service, deliver a Broadcast, pass data between components. Explicit (specific class) or Implicit (general action).
Components (Activity, Service) ને start/communicate message object. putExtra() = data pass. Explicit = specific class. Implicit = general action (share, open URL).
ExampleIntent i = new Intent(this, Next.class); i.putExtra("key","val"); startActivity(i);
Back StackCONCEPT
A stack of activities in the order they were opened. When you press Back, the top activity is popped. Launch modes (standard, singleTop, singleTask, singleInstance) control how activities are added to this stack.
Activities LIFO stack. Back button = top activity pop. Launch modes stack behaviour control. finish() = current activity pop.
BundleCONCEPT
A key-value container for passing data. Used to: pass data between activities (via Intent), save/restore UI state (onSaveInstanceState → Bundle → onCreate). Stores primitives, strings, arrays.
Key-value data container. Activities વચ્ચે data pass. onSaveInstanceState — UI state Bundle માં save. onCreate — Bundle restore. Screen rotation survive.
ContextCONCEPT
Provides access to the global application environment. Context is needed to access resources, file system, databases, system services (like Toast, getting LayoutInflater). Activity, Service, Application are all types of Context.
Global app environment access. Resources, file, DB, system services access Context વડે. Toast, LayoutInflater, getSystemService() — Context needed. Activity = Context type.
PermissionCONCEPT
Access control for sensitive device features. Normal permissions (INTERNET) = auto-granted. Dangerous permissions (CAMERA, LOCATION) = require runtime user approval since Android 6.0 Marshmallow. Declared in AndroidManifest.xml.
Sensitive features access control. Normal = auto grant. Dangerous (Camera, Location) = Android 6.0+ user approval. AndroidManifest.xml declare.
Main ThreadCONCEPT
The primary thread that handles UI rendering and user events. Never do heavy work (network, DB) on the main thread — it will freeze the UI (ANR error). Use background threads for heavy tasks.
UI render + user events handle main thread. Network/DB main thread પર ન કરો — UI freeze (ANR). Background thread/coroutine use.
ANRCONCEPT
Full FormApplication Not Responding
Error dialog shown when main thread is blocked for too long (5+ seconds). Caused by doing heavy work on the main/UI thread. Solution: move work to background thread.
Main thread block (5+ sec) → ANR dialog. Heavy work (network, DB) main thread = ANR. Background thread use.
Adapter PatternCONCEPT
Design pattern that separates data from UI in lists. The Adapter takes data (array, database) and creates View objects for each item. Used by ListView, RecyclerView, GridView, Spinner.
Data (array/DB) ↔ View (ListView/RecyclerView) bridge. Data + UI ને separate. Adapter = getView()/onBindViewHolder() per item.
Material DesignCONCEPT
Google's design language for Android introduced in Android 5.0 Lollipop. Based on physical metaphors like paper and ink. Includes elevation, shadows, animations, CardView, FAB, Snackbar, AppBarLayout.
Android 5.0 Lollipop માં introduce. Google's design system. Elevation, shadow, animation, CardView, FAB, Snackbar — Material Design components.
TransactionCONCEPT
A group of database operations treated as a single unit. Either ALL succeed (commit) or ALL fail (rollback). Used for bulk inserts — dramatically improves performance. beginTransaction() → setTransactionSuccessful() → endTransaction().
DB operations group — all succeed ya all fail. Bulk insert faster. beginTransaction() → setTransactionSuccessful() → endTransaction(). Rollback on failure.
FlowbeginTransaction() → operations → setTransactionSuccessful() → endTransaction()
CursorCONCEPT
Holds the results of a database query. Acts like a pointer that you move through rows. Key methods: moveToFirst(), moveToNext(), getCount(), getString(), getColumnIndex(), close().
DB query result hold. Row-by-row pointer. moveToFirst() → loop → moveToNext(). getString(getColumnIndex("col")) data read. close() always.
Patternif(cursor.moveToFirst()) { do { getString(idx); } while(cursor.moveToNext()); }
ViewHolderCONCEPT
Design pattern that caches View references for RecyclerView rows. Avoids repeated expensive findViewById() calls during scrolling. RecyclerView makes ViewHolder MANDATORY. Greatly improves list scroll performance.
RecyclerView row ના views cache. Scrolling time repeated findViewById() avoid. RecyclerView ViewHolder mandatory. Smooth scrolling.
Resource QualifierCONCEPT
Folder suffixes that provide different resources for different device configurations. -land (landscape), -sw600dp (tablet), -night (dark theme), -en (English). Enables adaptive UI without hardcoding.
Different screen/config માટે different resources. res/layout-land/ = landscape. res/values-night/ = dark theme. Adaptive UI = resource qualifiers.
Open SourceCONCEPT
Software whose source code is publicly available for anyone to view, modify, and distribute. Android is open source — maintained by Google under AOSP (Android Open Source Project). Anyone can modify it.
Source code publicly available. Android open source — Google AOSP maintain. Manufacturers customize (Samsung, MIUI). Free to use/modify.
Modal vs Non-ModalCONCEPT
Modal Dialog: Blocks all background interaction until dismissed. User must respond before continuing. Non-Modal: Does NOT block background. User can interact with both. Toast is non-modal.
Modal = background block — user respond જ કરવો. Non-modal = background interact possible. Toast = non-modal. AlertDialog = modal.
Event-DrivenCONCEPT
Programming model where execution flows based on events (user actions, system signals). Android is event-driven — button click, touch, lifecycle callbacks are all events. Listeners/callbacks handle events.
Program events (click, touch, lifecycle) based ચાલે. Android event-driven model. OnClickListener, OnTouchListener, lifecycle callbacks — events.

Important Classes
મહત્વના Classes
ActivityCLASS
A single screen with a user interface. The entry point of an app. Base class: AppCompatActivity. One activity = one screen. Has its own lifecycle (onCreate → onDestroy).
App ની single screen + UI. App ની entry point. AppCompatActivity extend. Lifecycle: onCreate → onDestroy. setContentView() UI load.
Classpublic class MainActivity extends AppCompatActivity { }
ServiceCLASS
A component that runs in the background WITHOUT a UI. Used for long-running operations: playing music, syncing data, downloading files. Has no visual interface.
Background task — UI નહિ. Music play, data sync, download. Background process. Service = no UI. Activity = UI.
FragmentCLASS
A reusable UI portion that lives inside an Activity. Has its own lifecycle. Managed by FragmentManager. Used for multi-pane layouts, tabs, ViewPager pages.
Activity ની અંદર reusable UI portion. Own lifecycle. FragmentManager manage. Tabs, multi-pane layout, ViewPager. Reuse possible.
BroadcastReceiverCLASS
Responds to system-wide broadcast announcements. No UI. Examples: SMS received, device booted, battery low, network changed. Register in Manifest or dynamically.
System broadcast receive. UI નહિ. SMS, boot, battery low, network change events. Manifest માં register.
ContentProviderCLASS
Provides data sharing between apps. Access to contacts, media, files through a standard URI interface. Android's contacts and media use ContentProviders.
Apps વચ્ચે data share. Contacts, media access. URI (Content URI) identify. Standard interface.
SQLiteOpenHelperCLASS
Helper class to manage SQLite database creation and versioning. Override: onCreate() — first-time database creation. onUpgrade() — called when DB version changes.
SQLite DB manage helper. onCreate() = DB first create. onUpgrade() = version change. getWritableDatabase() / getReadableDatabase().
Must OverrideonCreate(db) — first time | onUpgrade(db, old, new) — version change
SQLiteDatabaseCLASS
Represents an open database connection. Provides CRUD methods: insert(), update(), delete(), query(), rawQuery(), execSQL(). Get via getWritableDatabase() or getReadableDatabase().
Open DB connection represent. CRUD: insert(), update(), delete(), query(), rawQuery(), execSQL(). Write: getWritableDatabase(). Read: getReadableDatabase().
ContentValuesCLASS
A key-value container for storing column-value pairs for INSERT or UPDATE operations. Use put("column_name", value) to add each column's value.
Column-value pairs store. INSERT/UPDATE operations. put("name", "Raj") — column value add. insert(table, null, values).
UsageContentValues v = new ContentValues(); v.put("name","Raj"); db.insert("users",null,v);
LayoutInflaterCLASS
Converts an XML layout file into a View object at runtime. Used in: Adapters (getView), Fragment onCreateView(), Custom Dialogs. inflate() method does the conversion.
XML layout → View object runtime. Adapter getView(), Fragment onCreateView(), Custom dialog. inflate(R.layout.row, parent, false).
CanvasCLASS
The drawing surface for custom View graphics. In onDraw(Canvas canvas), use Canvas to draw shapes, bitmaps, text. Works together with Paint (style/color).
Custom View graphics drawing surface. onDraw() override. drawCircle(), drawRect(), drawText(), drawBitmap(). Paint = style/color.
PaintCLASS
Holds style and color information for drawing operations on a Canvas. Set color, stroke width, text size, fill vs stroke, anti-alias. Used together with Canvas.
Canvas drawing style + color. setColor(), setStrokeWidth(), setTextSize(), setStyle(FILL/STROKE). Canvas + Paint = custom graphics.
GestureDetectorCLASS
Detects common gestures: single tap, double tap, long press, scroll, fling (swipe). Override onTouchEvent() in your View and pass MotionEvent to GestureDetector.
Common gestures detect — tap, double tap, long press, scroll, fling/swipe. onTouchEvent() → GestureDetector.onTouchEvent(event).
ObjectAnimatorCLASS
Animates properties of any object (View). Can animate: alpha (transparency), rotation, translationX/Y, scaleX/Y. Part of the property animation system.
View properties animate — alpha, rotation, translationX/Y, scaleX/Y. ofFloat(view, "alpha", 0f, 1f). setDuration().start().
ExampleObjectAnimator.ofFloat(view, "alpha", 1f, 0f).setDuration(500).start();
FragmentManagerCLASS
Manages Fragment transactions (add, replace, remove) dynamically at runtime. Use getSupportFragmentManager() in Activity. beginTransaction().replace().commit().
Runtime fragments dynamically manage. add(), replace(), remove(). getSupportFragmentManager().beginTransaction().replace(R.id.container, fragment).commit().
PopupWindowCLASS
Manages floating popup UI windows that appear over the current content. Can contain any View layout. Custom dropdowns, tooltips, mini-menus.
Floating popup window manage. Custom view layout. Content ઉપર overlay. Custom dropdown, tooltip, mini-menu.
MotionLayoutCLASS
Extends ConstraintLayout to support complex constraint-based animations. Define start/end ConstraintSets in XML with a MotionScene. Google's most powerful UI animation tool.
ConstraintLayout extend — complex animations. Start/end ConstraintSet XML. MotionScene define. Most powerful Android animation.
SnackbarCLASS
Material Design replacement for Toast. Shows at the bottom of screen with optional action button (like UNDO). Lifecycle-aware, can be dismissed by swipe. Better than Toast for action-based feedback.
Toast નો Material Design replacement. Bottom of screen. Action button support (UNDO). Swipe dismiss. Lifecycle-aware. Action feedback = Snackbar.
CodeSnackbar.make(view, "Deleted", Snackbar.LENGTH_SHORT).setAction("UNDO", v->{}).show();
BitmapFactoryCLASS
Efficiently creates Bitmap objects from various sources: files, resources, byte arrays, streams. decodeResource(), decodeFile(), decodeStream() are key methods.
Bitmap images load. File, resource, stream থেকে. decodeResource(), decodeFile(). Image efficient load.

Important Methods
મહત્વની Methods
onCreate()METHOD
Called when Activity is created for the FIRST TIME. Called only once per lifecycle. Do here: setContentView(), findViewById(), initialize variables, setup UI.
Activity FIRST CREATE — once call. setContentView(), findViewById(), UI init. Lifecycle start.
UsesetContentView(R.layout.main); btn = findViewById(R.id.button);
onStart()METHOD
Called when Activity becomes VISIBLE to the user (after onCreate or onRestart). The Activity is visible but not yet interactive. Register listeners, load fresh data.
Activity VISIBLE but not interactive. onCreate/onRestart પછી. Listeners register, fresh data load.
onResume()METHOD
Called when Activity comes to the FOREGROUND and is fully interactive. HIGHEST PRIORITY state. Do here: start animations, camera, GPS, resume paused work. Called when returning from onPause.
Activity FOREGROUND + fully interactive. HIGHEST priority state. Animation start, camera/GPS resume. onPause → activity visible → onResume.
onPause()METHOD
Called when Activity is PARTIALLY HIDDEN (another activity appears on top, dialog shows). Activity is still VISIBLE. Do here: pause animations, release camera, save unsaved data quickly.
Activity PARTIALLY hidden — still visible. Dialog/another activity on top. Camera/GPS release. Unsaved data quick save. Short execution time.
onStop()METHOD
Called when Activity is COMPLETELY HIDDEN (user navigated to another app/home). Not visible at all. Do here: stop heavy operations, save to DB, unregister listeners.
Activity COMPLETELY HIDDEN — not visible. User navigated away/home. Heavy operations stop. DB save. Listeners unregister.
onRestart()METHOD
Called BEFORE onStart() when an Activity is returning from the STOPPED state (e.g., user presses back from another app). Opportunity to refresh data that may have changed.
STOPPED state → return — onStart() BEFORE call. Stopped activity back. Data refresh opportunity. onRestart → onStart → onResume sequence.
onDestroy()METHOD
Called right before Activity is DESTROYED. Triggers when: user presses Back, finish() is called, or system kills it. Do here: release all resources, close DB, unregister.
Activity DESTROY — back press / finish() / system kill. All resources release. DB close. Final cleanup.
onSaveInstanceState()METHOD
Called BEFORE Activity may be destroyed (screen rotation, system kills it). Save UI state (EditText values, scroll position, etc.) into the Bundle parameter. Restored in onCreate or onRestoreInstanceState.
Destroy/rotation BEFORE — UI state Bundle save. EditText, scroll position. onCreate(savedInstanceState) restore. Screen rotation survive.
CodeoutState.putString("name", editText.getText().toString());
setContentView()METHOD
Loads a layout XML file into the Activity and sets it as the screen content. Called in onCreate(). Parameter is the layout resource ID.
Activity ની screen content set. XML layout load. onCreate() માં call. R.layout.activity_main parameter.
UsagesetContentView(R.layout.activity_main);
findViewById()METHOD
Finds a View in the layout by its ID (android:id). Returns the View reference to control it programmatically. Call after setContentView().
android:id વડે View find. setContentView() પછી. Button, TextView, EditText reference get. Java code connect XML.
UsageButton btn = findViewById(R.id.myButton);
startActivity()METHOD
Launches another Activity. Pass an Intent as parameter. Adds the new Activity to the Back Stack. If you need a result back, use startActivityForResult() (deprecated → use ActivityResultLauncher).
Another Activity launch. Intent parameter. Back Stack push. Data pass = intent.putExtra().
CodestartActivity(new Intent(this, TargetActivity.class));
finish()METHOD
Closes the current Activity and removes it from the Back Stack. Triggers onPause() → onStop() → onDestroy() sequence. Similar to user pressing Back button.
Current Activity close. Back Stack pop. onPause → onStop → onDestroy trigger. Back button press same.
invalidate()METHOD
Forces a View to be redrawn on the next UI frame. Calls onDraw() again. Use when you've changed data in a custom View and need to update the display.
View redraw force. onDraw() again call. Custom View data change → invalidate() → UI update.
inflate()METHOD
Converts an XML layout file into a View object at runtime. Used by LayoutInflater. Essential in Adapters (create row views), Fragments (onCreateView), Custom Dialogs.
XML layout → View object. LayoutInflater.inflate(). Adapter row, Fragment onCreateView, Custom dialog — essential.
notifyDataSetChanged()METHOD
Tells the Adapter that data has changed — ListView/RecyclerView should refresh all its views. Call after adding, removing, or modifying data in the adapter's dataset.
Data change → list refresh. Adapter dataset modify → call. ListView/RecyclerView update display.
execSQL() / rawQuery()METHOD
execSQL(): Executes any SQL statement that doesn't return data (CREATE TABLE, DROP TABLE, INSERT without return). rawQuery(): Executes SELECT queries and returns a Cursor.
execSQL() = result ન return SQL (CREATE, DROP). rawQuery() = SELECT → Cursor return. SQL execute methods.
Usagedb.execSQL("CREATE TABLE t(id INT)"); | db.rawQuery("SELECT * FROM t", null);
moveToFirst() / moveToNext()METHOD
moveToFirst(): Positions cursor at the first row. Returns false if empty. moveToNext(): Advances cursor to next row. Use in a loop to iterate all rows.
moveToFirst() = first row. Empty = false. moveToNext() = next row loop. Cursor iteration pattern.
beginTransaction() / endTransaction()METHOD
beginTransaction() — starts a transaction. setTransactionSuccessful() — marks it as commit. endTransaction() — commits or rolls back. Essential for bulk inserts (major performance improvement).
beginTransaction() start → setTransactionSuccessful() commit mark → endTransaction() commit/rollback. Bulk insert performance.

Layouts — Types & Meanings
Layouts — પ્રકારો
LinearLayoutLAYOUT
Arranges children in a single row (horizontal) or column (vertical). orientation="horizontal" / "vertical". Simple and common. Avoid deep nesting — use ConstraintLayout.
Children vertical/horizontal line. orientation attribute. Simple forms. Deep nesting avoid — ConstraintLayout better.
RelativeLayoutLAYOUT
Positions views relative to each other or relative to the parent layout. Attributes: layout_below, layout_toRightOf, layout_centerInParent, layout_alignParentTop.
Views one another relative position. layout_below, layout_toRightOf, layout_centerInParent attributes. Each other reference.
ConstraintLayoutLAYOUT
The most powerful and RECOMMENDED layout. Positions views using constraints to parent or sibling views. Flat hierarchy (no nesting needed) = best performance. Best for responsive UI design.
Most powerful + RECOMMENDED. Constraints vade position. Flat hierarchy = performance best. Responsive UI. Nesting avoid.
FrameLayoutLAYOUT
Designed to hold a SINGLE child, but multiple children can be stacked (overlapping). Each child is drawn on top of the previous one. Used as Fragment containers, splash screens, overlay views.
Children overlap/stack on top. Fragment container, splash screen, overlay. Single child ideal but multiple stack possible.
TableLayoutLAYOUT
Arranges views in rows and columns like an HTML table. TableRow for each row. Each child in TableRow = one column. Good for structured data entry forms.
Rows + columns table. TableRow = each row. Forms, structured data entry. HTML table like.
GridLayoutLAYOUT
Places views in a flexible grid. Children can span multiple rows/columns. Unlike TableLayout, rows/columns don't need to be uniform. Used for icon grids, image galleries.
Flexible grid. Span multiple rows/columns. Icon grid, image gallery. GridLayoutManager (RecyclerView) different.
AbsoluteLayoutLAYOUT
⚠️ DEPRECATED. Positions views using absolute x/y pixel coordinates. Does NOT adapt to different screen sizes. Avoid completely — use ConstraintLayout instead.
⚠️ DEPRECATED — avoid. Fixed x/y coordinates. Different screens adapt ન thay. ConstraintLayout use.
ScrollViewLAYOUT
A layout that allows its single child to scroll vertically. Can contain only ONE direct child (usually a LinearLayout). HorizontalScrollView for horizontal scrolling.
Vertical scroll. Single child (usually LinearLayout). Long content. HorizontalScrollView = horizontal.
match_parentATTRIBUTE
Layout attribute value that makes a View expand to fill its parent's available size. Previously called fill_parent. Used for android:layout_width and android:layout_height.
Parent size fill. width/height = match_parent = parent full occupy. Previously fill_parent.
wrap_contentATTRIBUTE
Layout attribute value that makes a View just large enough to fit its content. A Button with wrap_content is only as wide as its text. No extra space.
Content size = View size. Button text → just fit. Extra space no. Content-based size.

UI Components & Special Terms
UI Components અને ખ્યાલો
